You already have almost exactly the shell you’re describing in `AdminPage.tsx` / `CreatorPage.tsx` plus `UnifiedNavBar` and `SystemsAIPanel`; the cleanest move is to standardize on that pattern and reuse it everywhere.[^1]

## 1. One-window shell layout

Replace your current page wrappers for Admin/Creator/System AI with this unified shell so the header is permanent and the content pane always fits the viewport with minimal scroll.

**`src/components/AppShell.tsx`** (new)

```tsx
import React, { ReactNode } from "react";
import Header from "./Header";
import UnifiedNavBar from "./UnifiedNavBar";
import "./Layout.css"; // you already have this

interface AppShellProps {
  children: ReactNode;
  activeTab: string;
  onTabChange: (id: string) => void;
  headerTitle: string;
}

export function AppShell({ children, activeTab, onTabChange, headerTitle }: AppShellProps) {
  return (
    <div className="min-h-screen flex flex-col bg-slate-950 text-slate-100">
      {/* Permanent header */}
      <header className="sticky top-0 z-50 border-b border-slate-800 bg-slate-950/95 backdrop-blur">
        <div className="max-w-[1800px] mx-auto px-4 py-3 flex items-center justify-between">
          <div className="flex items-center gap-3">
            <div className="h-9 w-9 rounded-full bg-gradient-to-br from-cyan-400 to-blue-500 flex items-center justify-center text-xs font-bold tracking-tight">
              NC
            </div>
            <div className="flex flex-col">
              <span className="text-xs text-slate-400 tracking-[0.2em] uppercase">
                NeuroCore
              </span>
              <span className="text-sm font-semibold tracking-wide">
                {headerTitle}
              </span>
            </div>
          </div>

          {/* main menu / exit slot – reuse what you already show in Header */}
          <Header />
        </div>

        {/* Horizontal menu bar just under header */}
        <div className="border-t border-slate-800">
          <div className="max-w-[1800px] mx-auto px-4 py-2">
            <UnifiedNavBar activeTab={activeTab} onTabChange={onTabChange} />
          </div>
        </div>
      </header>

      {/* Single content window below header/menu – constrained height */}
      <main className="flex-1">
        <div className="max-w-[1800px] mx-auto px-4 py-4">
          <div
            className="rounded-2xl border border-slate-800 bg-slate-950/80 shadow-[0_0_40px_rgba(15,23,42,0.75)] overflow-hidden"
            style={{ minHeight: "calc(100vh - 120px)" }}
          >
            <div className="h-full overflow-y-auto" style={{ maxHeight: "calc(100vh - 140px)" }}>
              {children}
            </div>
          </div>
        </div>
      </main>
    </div>
  );
}
```

This keeps one overall **window** that fills the screen, header/menu fixed, and the inner pane is the only scroll area.[^1]

## 4. Tight, low-scroll sizing

To keep scrolling minimal and unified:

- Rely on the **inner container** in `AppShell` (`overflow-y-auto` + `maxHeight: calc(100vh - 140px)`).
- In heavy panels, avoid nested `h-600px` etc.; switch to `h-full` so they expand inside the content pane and scroll there (for example, update `MaintenancePanel` root div from `h-[600px]` to `h-full`).[^1]

Example tweak inside `MaintenancePanel`:

```tsx
// old:
<div className="bg-slate-900/50 rounded-xl border border-slate-800 h-[600px] flex flex-col">

// new:
<div className="bg-slate-900/50 rounded-xl border border-slate-800 h-full min-h-[400px] flex flex-col">
```

Do the same kind of change for any component that hardcodes big pixel heights so the overall **one-window** layout stays tight and consistent.
